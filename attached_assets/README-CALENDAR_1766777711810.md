
# Calendar (Scheduling Reference Mirror)

## 1. Purpose
The Calendar tab is a **high-fidelity operational reference view** of the business schedule. It mirrors the **Google Calendar** source of truth, providing a unified timeline of installations, site visits, and client meetings linked to CRM records.

## 2. Core Authority Rules
*   **Google Calendar is Master**: SmartClicks does NOT own scheduling logic, availability checks, or conflict resolution.
*   **No Conflict Detection**: All scheduling disputes must be resolved in the authority system (GCal).
*   **Read-Only Mirror**: No event creation, editing, or deletion is supported inside the CRM tab.

## 3. Who Uses This
*   **Dispatchers**: To verify team availability and location status.
*   **Technicians**: To view their daily route and site details.

## 4. Sync & Refresh Model
*   **Fetch on Load**: Calendar data is retrieved immediately when the tab is accessed.
*   **Manual Refresh**: Data updates **ONLY** when the user clicks the "Refresh" button.
*   **No Background Polling**: The system does NOT auto-poll to prevent API quota abuse.
*   **Rate Limiting**: Refresh button is debounced (5s cooldown) to prevent hammering.

## 5. UI Layout & Visual Logic
*   **Views**: Day, Week, Month.
*   **Status Indicators**:
    *   **Imminent**: Starts within 60 mins (Amber).
    *   **Active**: Current time is between Start/End (Green/Pulse).
    *   **Completed**: End time passed (Gray).
*   **Overlaps**: Rendered side-by-side. No conflict warnings are generated by the CRM.

## 6. Timezone & Filtering
*   **Business Timezone**: All events render in the business's master timezone (e.g., PST). No per-user overrides.
*   **Linked Events**: By default, shows events with `entity_id` tags (Jobs, Deals).
*   **Unlinked Events**: Toggle available to show "All GCal Events" (Lunch, Holidays).

## 7. Click-by-Click Behavior
1.  **View**: User toggles Day/Week view.
2.  **Refresh**: User clicks Refresh icon to pull latest GCal state.
3.  **Action**: Clicking an event card opens **Google Calendar** in a new tab for editing.
    *   *Constraint*: The CRM does not open an edit modal.

## 8. Backend Expectations
*   **API**: Direct integration with Google Calendar API.
*   **Data Model**: Mapped from GCal Event Object -> CRM Calendar Card.
*   **Linkage**: Backend parses event descriptions to find `JOB-XXXX` tags for linking.

## 9. Edge Cases & Constraints
*   **API Failure**: If GCal is down, show a "Retry" banner, not a white screen.
*   **Broken Links**: If a CRM ID is invalid, the event still shows but is marked "Unlinked".
*   **Auth**: If Google Token expires, prompt for Marketplace re-auth.

## 10. What This Tab Is NOT
*   ❌ **A Scheduler**: You cannot drag-and-drop to reschedule.
*   ❌ **A Conflict Resolver**: It does not warn about double-bookings.
*   ❌ **A Write Surface**: It never writes to Google Calendar.
