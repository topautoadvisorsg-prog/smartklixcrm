
# Automation Ledger (Unified Event Transcript)

## 1️⃣ Purpose
The Ledger is the **Immutable Court Transcript** of the system. It records **events**, not just outcomes.
It enforces a strict **Chain of Custody** for every action.

**Crucial Architecture Rule**:
The Ledger is **Event-Based**. A single business workflow generates a **Chain of Ledger Entries**.

## 2️⃣ The Event Chain Model (Traceability)
Every workflow is assigned a unique `Trace ID`. The Ledger records the lifecycle across three authorities:

### Phase 1: Input & Proposal
*   **Actor**: ActionAI CRM (AI)
*   **Event Type**: `AI_PROPOSAL_CREATED`
*   **Data**: Raw input source, generated proposal payload, intent.

### Phase 2: Governance & Validation
*   **Actor**: Master Architect (AI)
*   **Event Type**: `AI_REVIEW_DECISION`
*   **Data**: Policy checks passed/failed, automated decision (Approve/Reject).

### Phase 3: Authority & Execution
*   **Actor**: Human Operator
*   **Event Type**: `HUMAN_EXECUTION_DECISION`
*   **Data**: The physical button press, user identity, and final execution state.

## 3️⃣ UI Layout & Components
*   **Trace View**: Grouping events by their Workflow ID.
*   **Columns**: `Trace ID`, `Event Type`, `Actor`, `Details`, `Timestamp`, `Hash`.
*   **Integrity Check**: Visual indicator that the hash chain is unbroken.

## 4️⃣ Backend Expectations
*   **Append-Only**: Database **MUST** disallow updates or deletes.
*   **Cryptographic Chaining**: Each entry contains a hash of the previous entry to prevent tampering.
