Required logic for completed calls

Webhook receives the call_completed payload

Call POST /api/contacts/create with at least:

{
  "phone": "{{ $json.from_number }}",
  "name": "{{ $json.caller_name || '' }}"
}


Use the returned id as contactId

If details.job_type is present

Call POST /api/jobs/create with:

contactId from above

jobType = details.job_type

plus other detail fields (propertyType, address, city, budget, preferredTime, notes)
Else

Call POST /api/leads/create with:

contactId

reason = reason from payload

summary = ai_summary if present

Call POST /api/activity-log/write with:

contactId

type: "call"

direction: "inbound"

summary: use ai_summary if available, otherwise a simple fallback like "Inbound call from {from_number}"

metadata: include call_id, job_type, budget, etc

Optionally call POST /api/events/update with eventId = call_id, eventType = "call_completed", status, and a result object
