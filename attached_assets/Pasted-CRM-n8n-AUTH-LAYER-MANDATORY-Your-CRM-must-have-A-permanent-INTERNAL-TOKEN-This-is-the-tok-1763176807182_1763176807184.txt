CRM → n8n AUTH LAYER (MANDATORY)

Your CRM must have:

✔ A permanent INTERNAL_TOKEN

This is the token n8n uses when it calls your CRM.

Your dev must build:

ENV variable: N8N_INTERNAL_TOKEN

Middleware that checks this on all protected endpoints

Header format:

Authorization: Bearer <N8N_INTERNAL_TOKEN>

✔ Health check endpoint

So n8n can test the connection.

Example:

GET /internal/health

✔ Secure endpoint for creating/updating contacts/jobs/etc

n8n will interact with:

/contacts/create

/contacts/update

/jobs/update-status

/estimates/create

/invoices/create

/payments/record

/activity-log/write

All must accept AUTH via your internal token.

2. CRM → n8n WEBHOOK SYSTEM

Your dev must build:

✔ A place in your CRM to configure:

N8N_BASE_WEBHOOK_URL

Your CRM will call n8n webhooks for events like:

new lead created

estimate sent

job completed

invoice overdue

missed call

etc

These POST calls MUST include:

event
data: {}
timestamp
signature (optional)


This gives n8n exactly what it needs to start flows.
