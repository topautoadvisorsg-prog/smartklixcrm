THERE ARE TWO SEPARATE CHATBOTS — COMPLETELY DIFFERENT ROLES
✅ 1. PUBLIC CHAT WIDGET (Customer-Facing)

Purpose: for leads, website visitors, customers.

This bot:

Lives on your public website

Collects lead info

Starts conversations

Responds to customer questions

Triggers N8N workflows (lead creation, follow-up sequences, automations, etc.)

Uses strictly limited data

Must NOT access your CRM internals

Must NOT control your systems

This bot = customer service + sales intake.

🧠 2. ADMIN INTELLIGENCE BOT (Internal CRM Chatbot)

Purpose: YOU + your team talk to it inside the CRM.

This bot:

Lives INSIDE your SaaS CRM dashboard

Has access to all internal data

Connects directly to:
✓ CRM database
✓ pgvector memory
✓ N8N automations
✓ Notion
✓ Stripe
✓ Contact records
✓ Leads
✓ Job information
✓ Any backend services

And it can:

Answer questions like ChatGPT

Run actions

Pull data

Update CRM

Create tasks

Trigger workflows

Query memory

Summarize conversations

Give analytics

Control systems through N8N

This bot = your company’s brain.

This is your “Master Architect” LLM.

🔍 WHY THE DISTINCTION MATTERS (Critical)

If the developer mixes these two up:

Your customer bot could access private internal data → security breach

Your admin bot could get limited like a customer bot → useless

N8N workflows will break → automation failures

Memory will store the wrong things → corruption

You absolutely MUST separate them.

This is not optional.
