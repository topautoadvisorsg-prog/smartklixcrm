Credential Management Responsibilities
Neo8 (n8n) Responsibilities:

You must configure and manage all execution-layer credentials in Neo8, including:
Stripe API key and webhook signing secret
Twilio credentials
SendGrid credentials
Google OAuth credentials
Redis credentials (if used)
OpenAI API key (if using AI nodes)
All these credentials are stored and managed exclusively within Neo8 (n8n).
Neo8 is responsible for all direct integrations with external services (Stripe, Twilio, Google, etc.).
Smart Klix CRM Responsibilities:

Does NOT store or manage any third-party secrets (Stripe, Twilio, etc.).
Does NOT call Stripe or other external APIs directly.
Does NOT validate Stripe (or other) webhooks.
Acts only as the system of record, sending intent to Neo8 and receiving status callbacks from Neo8.
A shared internal token (N8N_INTERNAL_TOKEN) is only required for live CRM callbacks, and can be added later.
What This Means for Your Setup
You can fully configure and test all external service credentials and workflow logic in Neo8 without waiting for the CRM token.
You should focus on:
Setting up and testing all external integrations (Stripe, Twilio, SendGrid, Google, etc.) in Neo8.
Mapping payloads and ensuring correct data flow between nodes.
Using test/sandbox environments for all external services.
You do NOT need to configure any CRM credentials or secrets at this stage.
You can use placeholder values or mock endpoints for CRM callbacks until the N8N_INTERNAL_TOKEN is provided.
Next Steps
Proceed with configuring all required credentials in Neo8 (n8n).
Set up and test all external service nodes using their respective test environments.
Prepare payload mappings and workflow logic.
When the N8N_INTERNAL_TOKEN is provided, add it to the relevant HTTP Request nodes for CRM callbacks.
