FULL PROJECT RESET — NEO8 WORKFLOWS (READ CAREFULLY)

We are restarting this project with a clean mental model.

This system is a multi-workflow automation architecture built in n8n (Neo8).
Neo8 is the central intake, routing, logging, and orchestration layer between:

CRM

Website widgets and forms

GPT / AI systems

AI receptionist (Twilio Voice)

SendGrid (email)

Stripe (payments later)

Google Workspace (Sheets, Docs, Calendar, etc.)

Rule:
Anything that creates, updates, or routes meaningful information must pass through Neo8 so it can be logged, normalized, reviewed, and approved.

🚪 WORKFLOW #1 — LEAD / DATA INTAKE FLOW (FIRST PRIORITY)

This is the first workflow and it is INTAKE ONLY.

The purpose of the Intake Flow is to capture incoming information, standardize it, and make it available to the rest of the system.

What “Intake” means in this system

Intake =
Receiving structured information from external systems and funneling it into Neo8 without making decisions yet.

This includes:

Leads

Contact details

Messages

Structured updates from GPT systems

Form submissions

AI-generated summaries or outputs

✅ SOURCES THAT FEED THIS INTAKE FLOW

This workflow must be able to accept input from:

Website chat / widget
→ Leads generated by users chatting on the site

Contact Us / Web forms
→ Standard form submissions

GPT / ChatGPT systems
→ Leads, SOPs, summaries, structured data, or updates
(Not conversations — outputs)

AI Receptionist (Phone via Twilio)
⚠️ IMPORTANT:

The AI receptionist itself is a separate workflow

This Intake Flow only has a handoff / intake node from that system

No phone logic lives here. No call handling lives here.

❌ WHAT THIS INTAKE FLOW DOES NOT DO

This is where confusion keeps happening, so read carefully.

This flow does NOT:

Run AI conversations

Make CRM decisions

Decide lead routing

Trigger follow-ups

Send SMS

Send emails

Handle payments

Execute sales logic

Those all belong to downstream workflows.

The Intake Flow’s job is to:

Receive

Normalize

Log

Tag

Pass forward

Nothing more.

🧩 CORE LOGIC INSIDE THE INTAKE FLOW

High-level logic only (no configuration here):

Receive incoming data

From any approved source

Validate minimum structure

Is this usable information?

Normalize data

Names, emails, phone numbers, tags, sources, timestamps

Source identification

Where did this come from? (widget, form, GPT, phone handoff, etc.)

Central logging

Store raw + normalized data for traceability

Prepare for downstream workflows

CRM sync

Google Sheets

Future automations

🧠 MASTER ARCHITECT — REQUIRED CHECKPOINT

This flow must include a visible Master Architect checkpoint.

This is non-negotiable.

Purpose:

Review intake structure

Approve changes

Validate assumptions

Authorize downstream automation

This can be:

A placeholder node

A manual review step

An approval flag

But it must be clearly labeled so no developer bypasses it.

Neo8 is not just a webhook catcher.

It is the system of record for automation logic.

If data touches:

CRM

Google Workspace

AI systems

Communication tools

Then Neo8 sees it first.
