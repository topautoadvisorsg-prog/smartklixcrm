Required validation rules (implement exactly)

timestamp must be valid ISO8601.

source.channel must match the allowed enum values in the envelope.

idempotency_key must be unique per tenant.

Required endpoint behavior

Endpoint path: /api/intake/lead (HTTP POST).

On validation error: respond 400 with JSON error explaining field(s).

On duplicate idempotency_key: respond 200 with { "status": "duplicate" }.

On success: respond 201 with { "status":"accepted", "contactId":"...", "outboxId":"..." }.

All successful persistence (contact create/link + outbox row) must be in a single DB transaction.
