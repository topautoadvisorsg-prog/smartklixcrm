The core misunderstanding (not your fault)

N8N_INTERNAL_TOKEN
CRM_BASE_URL

❌ Do NOT go in:

Credentials

Nodes

Workflow settings

“Complete these steps before executing workflow”

Any text field in the UI

✅ They ONLY exist as environment variables on the machine/container that runs n8n

n8n just reads them. It does not store them.

That’s why you feel like “there’s nowhere to put this info” — because there isn’t.

Think of it like this

n8n UI = dashboard

Environment variables = hidden configuration under the hood

Workflows can reference env vars, but never define them

So when a node uses:

{{$env.N8N_INTERNAL_TOKEN}}

n8n is basically saying:

“Hey operating system / container, do you have a variable named N8N_INTERNAL_TOKEN?”

If the OS says “no”, the workflow fails.

What “complete these steps before executing workflow” REALLY means

That message is not asking you to paste values somewhere.

It means:

“Stop. Go configure your runtime (Docker / server / terminal). Then come back.”

That wording is honestly misleading.

How you know you did it right (important)

After you set the variables and restart n8n, do this:

Open any node

In a field, temporarily type:

{{$env.CRM_BASE_URL}}

Click outside the field

If it shows the URL → ✅ you’re done
If it stays blank / errors → ❌ env vars not set or n8n not restarted

You can remove it afterward — this is just a test.

❌ Running n8n via Docker Desktop but editing a different compose file

❌ Using Windows and setting them in the wrong shell (cmd vs PowerShell)
