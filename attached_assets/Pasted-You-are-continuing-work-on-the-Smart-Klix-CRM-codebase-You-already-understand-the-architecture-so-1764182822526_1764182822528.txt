3.2 Internal voice orchestration endpoint, text only for now

We need a backend entrypoint that n8n can call when a voice turn happens, once Twilio is wired to n8n.

Create an endpoint like:

POST /api/voice/receptionist/turn

Request body should include:

channel: "voice"

caller_phone

conversation_id string, for multi turn calls

transcript string, caller text

Optional call_state JSON object for state machine info if needed

Implementation steps:

Load AI Receptionist config for the tenant

Build a structured context object for Master Architect with

channel = "voice"

caller info if contact exists

call state if provided

Call the Master Architect pipeline with that context

Respect the same Draft, Assist, Auto behaviors we already use

Draft, only reply text, no tool execution

Assist, queue tools into the assist queue, do not execute directly

Auto, execute allowed tools immediately, respecting tool permissions

Return a response to the caller, for example

{
  "reply_text": "Thank you, I have booked your appointment for tomorrow at 3 PM",
  "actions_taken": [
    {
      "tool": "schedule_appointment",
      "status": "queued",
      "mode": "assist"
    }
  ],
  "conversation_id": "voice_abc123",
  "metadata": {
    "handoff_suggested": false
  }
}


We are not doing STT or TTS inside this endpoint right now. That will live in n8n and Twilio.
