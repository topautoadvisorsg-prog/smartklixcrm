🧩 1. NEW DATA MODELS (ADD TO DATABASE)
A. Estimates

Create new table: Estimates

id (PK)
contact_id (FK)
job_id (FK, nullable until accepted)
status ENUM('draft','sent','accepted','rejected')
line_items JSONB
subtotal NUMERIC
tax_total NUMERIC
total_amount NUMERIC
valid_until TIMESTAMP NULL
created_at TIMESTAMP
updated_at TIMESTAMP

B. Invoices

Create new table: Invoices

id (PK)
job_id (FK)
contact_id (FK)
estimate_id (FK, nullable)
status ENUM('draft','sent','paid','overdue','void')
line_items JSONB
subtotal NUMERIC
tax_total NUMERIC
total_amount NUMERIC
issued_at TIMESTAMP NULL
due_at TIMESTAMP NULL
paid_at TIMESTAMP NULL
created_at TIMESTAMP
updated_at TIMESTAMP

C. Payments

Create new table: Payments

id (PK)
invoice_id (FK)
amount NUMERIC
method ENUM('cash','card','bank_transfer','other')
transaction_ref TEXT NULL
status ENUM('pending','completed','failed','refunded')
paid_at TIMESTAMP NULL
created_at TIMESTAMP

D. Extend Jobs table into full job ticket/work order

Add the following fields to existing Jobs table:

job_type ENUM('lead','job') DEFAULT 'lead'
status ENUM('lead_intake','estimate_sent','scheduled','in_progress','completed','invoiced','paid','cancelled')
job_number TEXT UNIQUE
scheduled_start TIMESTAMP NULL
scheduled_end TIMESTAMP NULL
closed_at TIMESTAMP NULL
assigned_techs JSONB (array of user IDs)
source_lead_id (FK nullable)
source_estimate_id (FK nullable)


This converts Jobs into full field-service job tickets without overhauling existing logic.

🧩 2. PIPELINE FLOW (REQUIRED SYSTEM LOGIC)

The platform must now support this mandatory pipeline:

Lead → Estimate → Job → In Progress → Completed → Invoice → Payment

Rules:

When a lead is ready for pricing, create an Estimate linked to the contact.

If accepted:

convert into a Job (job_type=job)

job.status = 'scheduled'

When technician or staff start work:

job.status = 'in_progress'

When finished:

job.status = 'completed'

closed_at = timestamp

When invoice created:

job.status = 'invoiced'

When payment recorded:

job.status = 'paid'

Status changes MUST log into AuditLog automatically.
