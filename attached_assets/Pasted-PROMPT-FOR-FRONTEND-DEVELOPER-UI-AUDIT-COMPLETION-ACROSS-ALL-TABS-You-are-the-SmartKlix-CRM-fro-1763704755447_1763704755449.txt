1. Global Layout & UX

Across the whole app:

Top bar

Show current user

Quick access to search (contacts / jobs)

Global notifications / toasts for success and errors

Sidebar

Clean nav with all core tabs:

Dashboard

Contacts

Jobs / Leads

Appointments / Calendar

Activity / Audit Log

AI Assist / Queue

Templates

Files

Settings

N8N / Integration Status

General rules

Every main action must give visible feedback

No blank screens, always show either data, “no results” empty state, or loading skeletons

All destructive actions require confirmation modals

Purpose: Give a quick overview of the business at a glance.

It must include:

Key metrics cards

New leads this week

Upcoming appointments

Open jobs by status

Missed calls (if available)

Recent activity feed

Show latest actions from audit log (who did what, when, on which contact/job)

Clickable links into contact / job detail pages

Quick actions

“Add contact”

“Create lead / job”

“Log call / note”

Everything here must display real data, no mock values.

Contacts list

Paginated table

Search by name, email, phone

Filters (tag, lead stage if available)

Contact detail view

Core info: name, phone, email, tags

Related jobs / leads list

Recent activity (notes, calls, SMS, email if tracked)

Files linked to this contact

Action buttons:

Send SMS

Send Email

Create Job / Lead

Log Note

Log Call

Create / Edit contact

Proper validation and error display

No silent failures

Purpose: Track work and sales pipeline.

Jobs / leads list

Job detail

Linked contact

Description / scope

Status and ability to change it

Appointment(s) linked

Notes

Files

Internal activity

Actions

Purpose: Manage scheduling.

It must support:

Calendar view

Appointment list

Show contact, job, date/time, status (confirmed / pending / completed)

Create / edit appointment

Choose date and time

Save to backend

Display validation errors correctly

Actions

Reschedule

Mark as completed

Cancel

Purpose: Show a history of what happened in the system.

List of actions

Timestamp

Actor (user or system/AI)

Action summary

Target entity (contact, job, appointment)

Filters

Date range

Type (note, job update, lead created, SMS, etc. if available)

Purpose: Review and approve AI suggested actions.

It must support:

Queue list

Summary of suggestion

Target (contact / job)

Type of action (send SMS, follow up, update status, etc.)

Mode (Draft / Assist / Auto if that’s surfaced)

Detail panel

Full suggested text or action details

Any context the AI used

Actions

Reject → marks as rejected

Mark as done / resolved

No item should be “clickable” without doing anything. Wire it to the real endpoints already defined.

Template list

Name

Type (SMS / Email)

Filename

Uploaded at

Basic business info

Across all tabs:
