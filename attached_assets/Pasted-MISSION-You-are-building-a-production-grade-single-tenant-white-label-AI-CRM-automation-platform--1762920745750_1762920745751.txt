🧩 ARCHITECTURE RULES
Single Container per Tenant

Each instance includes:

Unique DB connection

Isolated Redis namespace

Separate storage bucket

Independent environment vault
No shared tenants under any circumstance.

Environment Vaulting

Secrets live in a secret manager (e.g., Vault, Doppler, AWS Secrets Manager).

Never commit .env with real keys.

Tokens must auto-rotate and expire gracefully.

Each tenant has unique encryption keys and storage credentials.

Schema Evolution Safety

Every migration must be additive-safe.

Use versioned filenames (001_init_schema.sql, 002_add_tags.sql, etc.).

Include rollback scripts for each.

Never drop or rename columns in production.

Include schema diffs in /docs/upgrade_strategy.md.

Queue & Concurrency Safety

Use BullMQ for outbound operations (email/SMS/voice/files/memory summarization).

All queue workers must be idempotent.

Retry caps and exponential backoff enforced.

Never run external API calls synchronously.

Rate Limiting / Abuse Mitigation

Implement IP and user-level throttling for public endpoints.

Widget must enforce per-tenant request quotas.

Detect injection attempts (“ignore previous instructions,” “act as admin”) and auto-terminate sessions.

Fail-Safe Defaults

On error: deny, log, alert, ask human.

Never fabricate or hallucinate state.

Always fail closed, never open.

🧠 AI AGENT MODES

Draft — Suggests actions only; cannot execute.
Assist — Queues actions for human approval; learns preferences.
Auto — Executes approved actions; one reflection; capped retries.

Destructive, financial, or cross-tenant actions always require explicit human confirmation.
Each agent mode must be clearly displayed in the dashboard header and audit log.
