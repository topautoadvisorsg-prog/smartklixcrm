---

## 2) CLIENT ID FORMAT (LOCK THIS IN)

We are intentionally keeping Client IDs simple.

**Format:**

```
<first_name>_<4-digit-random>
```

**Examples:**

* joe_4837
* maria_9021
* alex_1174

Rules:

* First name only (lowercase or normalized)
* One underscore `_`
* Four random digits
* No UUIDs
* No long hashes
* No 8–16 digit nonsense

This ID is:

* Human-readable
* Stable
* Used everywhere (CRM, messaging, ledger, Neo8)

## 3) WHATSAPP & SMS — PURPOSE

This is **NOT chat**
This is **NOT real-time conversation**

This is:

* Outbound outreach
* Follow-ups
* Transactional or semi-personal messaging
* Logged, reviewable, auditable actions

---

## 4) ACTIONAI AUTHORITY (WHATSAPP / SMS)

ActionAI **CAN**:

* Draft SMS / WhatsApp messages
* Propose outreach tied to a valid Client ID
* Ask clarifying questions *before drafting* if required data is missing

ActionAI **CANNOT**:

* Send messages directly
* Bypass review
* Message without a Client ID

All ActionAI-generated messages must:

1. Be tied to a valid Client ID
2. Enter the Review Queue
3. Be approved by a human
4. Be dispatched via Neo8
5. Be written to the ledger

---

## 5) HUMAN AUTHORITY

Humans:

* Can approve ActionAI-drafted messages
* Can manually draft messages (still requires Client ID)
* Can authorize dispatch

Humans still **do not** bypass:

* Neo8
* Ledger logging

---

## 6) DISPATCH FLOW (REQUIRED)

**No direct sends from CRM. Ever.**

Flow:

```
CRM UI
→ Review Queue
→ Authorization
→ Neo8
→ Provider (WhatsApp / SMS)
→ Ledger Write
```

---

## 7) LEDGER RULES (REMINDER)

Ledger entries are for **actions only**.

WRITE ledger entries for:

* SMS_DISPATCH_AUTHORIZED
* SMS_SENT
* SMS_DELIVERED
* SMS_FAILED
* WHATSAPP_DISPATCH_AUTHORIZED
* WHATSAPP_SENT
* WHATSAPP_DELIVERED
* WHATSAPP_FAILED

DO NOT write ledger entries for:

* Viewing a message
* Drafting text
* Asking questions
* Reading conversations

No Client ID = No Ledger entry.
