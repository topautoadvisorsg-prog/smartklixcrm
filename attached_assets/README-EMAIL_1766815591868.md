
# Email (Communication Dispatch & Intelligence)

## 1. Core Intent
The Email tab is a **Unified Communication Feed**, not a traditional productivity inbox.
Its primary purpose is **Visibility** and **Traceability** of all contact-linked communications.

It aggregates:
*   **Personal Communication** (via Gmail API)
*   **System Communication** (via SendGrid/Neo8)

## 2. The Identity Model (Critical Architecture)
The system enforces a strict boundary between **User** and **System** identities. This is a hard architectural constraint.

### A. User Identity (Gmail)
*   **Author**: The human operator (e.g., Sarah Architect).
*   **Provider**: Gmail API (OAuth).
*   **Context**: Personal, ad-hoc, free-text replies.
*   **Signature**: Injected automatically based on CRM profile.

### B. Company Identity (SendGrid)
*   **Author**: The System (SmartClicks).
*   **Provider**: SendGrid (via Neo8 Relay).
*   **Context**: Transactional, structural, templated (Invoices, Alerts).
*   **Constraint**: Users cannot free-type "System" emails. They must select approved templates.

### C. The Friction Rule
*   **Behavior**: Switching identities in the Compose Modal **clears the draft body**.
*   **Reason**: Prevents cross-contamination (e.g., sending a casual draft via the formal System channel).

## 3. Backend Contract (Non-Negotiable)
The Frontend CRM **NEVER** sends emails directly. All outbound dispatch actions are requests to the **Neo8 Engine**.

1.  **UI Action**: User clicks "Authorize Dispatch".
2.  **Payload**: UI constructs a standardized JSON payload (`target`, `body/template`, `identity_provider`).
3.  **Neo8**: Receives payload, validates permissions, routes to provider (Gmail/SendGrid), and logs the event to the **Ledger**.
4.  **Tracking**: Open/Click events are web-hooked back to Neo8, which updates the UI status badges.

## 4. Privacy & Filtering Rules
This tab is **NOT** a full mailbox mirror. It is a filtered operational view.

*   **Ingress Filter**: Emails are only displayed if they match an existing **Contact** or **Job** record in the CRM.
*   **Exclusion**: Private, personal, or unrelated emails in the connected Gmail account are **never** ingested or displayed.
*   **Governance**: This ensures the CRM remains an operational tool, not a personal data leak.

## 5. UI Layout & Interaction
*   **Sidebar**: "Communication Filters" with breakdown of Personal vs System message counts and "Ingress Filter Active" status.
*   **Feed**: Color-coded cards (Blue for Personal, Purple for System) showing status, sender, and preview.
*   **Compose Modal**:
    *   **Identity Switcher**: Toggle between Gmail (Free-text) and System (Template-only).
    *   **Gmail Mode**: Subject and Body inputs are editable.
    *   **System Mode**: Body input is replaced by a **Template Selector** and read-only payload preview.
    *   **Authorization**: Single "Authorize Dispatch" button triggers the Neo8 engine.

## 6. Explicit Non-Goals
*   ❌ **Inbox Zero**: This is an audit log, not a to-do list.
*   ❌ **Marketing Builder**: Bulk campaigns are managed in `Funnels` or `Social Planner`.
*   ❌ **Real-Time Chat**: Use `WhatsApp` for instant messaging.
*   ❌ **Auto-Send**: No background automation triggers here (those live in `ActionAI`).
