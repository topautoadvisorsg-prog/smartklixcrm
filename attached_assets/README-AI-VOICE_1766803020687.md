
# AI Voice (Manual Dispatch & Configuration)

## 1. Purpose
The AI Voice tab is a **manual dispatch and configuration interface** whose sole responsibility is to prepare and send outbound call instructions to the AI Voice Server. It acts as the staging ground for voice intent.

## 2. Engine Routing & Configuration (Mandatory)
The AI Voice tab **MUST** explicitly facilitate and document the routing selection for the voice mission.

*   **User Control**: The user must explicitly select between **Economic** and **Premium** engines via the UI.
*   **Visibility**: The selected engine (Economic vs. Premium) must be visible in the configuration panel.
*   **Payload Inclusion**: The engine selection is a hard requirement for the outbound payload sent to the AI Voice Server.
*   **Constraint**: This selection is **NOT** optional and is **NOT** implied. It is a strict configuration parameter defined by the human operator.

## 3. Human vs. Automation Boundary (Critical)
The system enforces a strict separation between **Human Definition** and **Server Execution**.

**The Human Operator Defines (Left Panel):**
*   **Call Intent** (e.g., "Schedule maintenance")
*   **Contact** (Who to call)
*   **Context** (Specific details or script notes)
*   **Engine Selection** (Economic or Premium)
*   **Authorization** (Permission to execute)

**The System Executes (Right Panel):**
*   The **AI Voice Server** performs all automation.
*   The **Log Feed** passively displays status (`In-Progress`, `Success`, `Failed`) returned by the server.

**Rule**: Automation begins **only** after instructions are received by the AI Voice Server.

## 4. Execution Flow
The only valid lifecycle for a voice mission is sequential and strictly ordered:

1.  **Human Definition**: Operator defines Intent, Context, Contact, and Engine in the AI Voice Tab (Left Panel).
2.  **Authorization**: Operator checks the box and clicks Dispatch.
3.  **Dispatch**: CRM sends the specific payload (including Engine selection) directly to the **AI Voice Server**.
4.  **Execution**: AI Voice Server receives the payload and executes the automated call.
5.  **Relay**: After the call, the AI Voice Server sends the outcome (success/failure, transcript, summary) back through **Neo8**.
6.  **Update**: CRM updates its source of truth based on the Neo8 relay, visible in the Right Panel.

## 5. Authorization Checkbox
The authorization checkbox in the UI is a **human confirmation of intent**.

*   It is **NOT** an automation trigger logic.
*   It is **NOT** an AI governance step.
*   It is **NOT** a safety review.

**Once checked and dispatched:**
*   The call **will** execute.
*   No additional logic or governance exists in this tab to stop it.

## 6. Post-Call Handling
The AI Voice tab acts as a passive viewer for completed actions.

*   The tab does **NOT** interpret results.
*   The tab does **NOT** retry calls.
*   The tab does **NOT** judge success or failure.
*   It only dispatches instructions and later displays returned outcomes from the source of truth.

## 7. What This Tab Is NOT
*   ❌ **Not an autonomous caller**
*   ❌ **Not where call automation logic lives**
*   ❌ **Not an AI decision-maker**
*   ❌ **Not responsible for call success or failure**
*   ❌ **Not a softphone** (No human speaking)
*   ❌ **Not a live listening post**
