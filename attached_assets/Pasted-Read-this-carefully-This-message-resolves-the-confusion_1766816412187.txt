## 1) EMAILS TAB — STRUCTURE (IMPORTANT)

There is **ONE main Emails tab**, with **SUB-TABS inside it**.

### Sub-Tab A: Emails (Personal Email)

Purpose:

* Human-sent, personal communication
* Uses Google / Gmail–connected accounts
* Free-text subject and body

Used for:

* 1:1 outreach
* Ad-hoc follow-ups
* Non-templated human communication

Authority:

* Human initiates
* ActionAI may DRAFT, but:

  * Must go through Review Queue
  * Must be approved before send
* Dispatch still goes through Neo8
* Ledger records DISPATCH actions only

---

### Sub-Tab B: Email Accounts (Company / System Email)

Purpose:

* System and business email identity
* Used for invoices, notifications, transactional messages

Provider:

* SendGrid / system provider
* Routed via Neo8

Used for:

* Templates
* Company-authoritative communication
* Non-free-text emails

Rules:

* No free-text editing at send time
* Template selection only
* Templates are pre-approved assets

Authority:

* Human or ActionAI can select template
* ActionAI proposals MUST go through Review Queue
* Human approval required before execution
* All sends dispatched via Neo8
* All dispatch events written to ledger

Templates ARE part of this scope.

## 4) ACTIONAI AUTHORITY

ActionAI:

* CAN draft emails
* CAN select templates
* CANNOT send directly
* EVERYTHING it does must:
  → Create a proposal
  → Enter Review Queue
  → Be approved
  → Then executed via Neo8

Humans:

* Can send personal email directly
* Company email still requires authorization + ledger

---

## 5) LEDGER RULES (REMINDER)

Ledger entries are for ACTIONS ONLY.

Write ledger entries for:

* EMAIL_DISPATCH_AUTHORIZED
* EMAIL_DISPATCH_SENT
* EMAIL_DELIVERED
* EMAIL_OPENED
* EMAIL_CLICKED
* EMAIL_FAILED

DO NOT write ledger entries for:

* Reading emails
* Viewing lists
* Searching
* Asking questions
