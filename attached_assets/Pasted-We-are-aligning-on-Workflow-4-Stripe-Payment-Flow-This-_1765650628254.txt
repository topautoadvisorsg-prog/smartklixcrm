Contact

Invoice

Log outcomes for humans

Failed / refunded payments → human review queue / task

🧠 Clarifying Architecture (Please Confirm)

Please confirm the following assumptions are correct:

Payments vs Invoices

Invoice = request for payment (“money owed”)

Payment = actual money movement event

One invoice can have multiple payment attempts

Payments tab is a normalized ledger, not Stripe itself

Stripe Responsibilities

Stripe handles:

Card input

SCA / 3DS

Retries (soft declines)

Payment intent lifecycle

CRM Responsibilities

CRM:

Creates payment links / invoices (via Stripe API)

Listens to Stripe events

Normalizes events into Payments tab

Stores NO sensitive payment data

Neo8 Responsibilities (Later)

Neo8 will:

React to CRM payment events

Log activity

Queue human follow-up when needed

Neo8 will not:

Process payments

Create Stripe objects directly

Handle card data
