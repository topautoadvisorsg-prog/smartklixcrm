FULL SYSTEM RE-AUDIT CHECKLIST

Scope: Intake → ActionAI Console → Review Queue → Ready Execution → Ledger
Goal: Prove the system behaves correctly every single time, with no silent paths

I. INTAKE AUDIT (CRITICAL – MUST BE FIRST)

Intake is the front door. If this is wrong, everything downstream is corrupted.

1️⃣ Intake Configuration Verification

Audit Questions

What endpoint receives intake payloads?

What sources are supported?

External GPTs

Widget

API calls

Manual operator intake

Required Proof

Show the intake endpoint

Show payload schema

Show source identifier (e.g. source = widget | gpt | operator)

2️⃣ Intake → Ledger (MANDATORY)

Expected Behavior

Every intake creates a ledger entry:

INTAKE_RECEIVED

includes source, entity (contact/company), timestamp

Fail Conditions

Intake creates a proposal without a ledger entry

Intake silently routes to ActionAI without trace
