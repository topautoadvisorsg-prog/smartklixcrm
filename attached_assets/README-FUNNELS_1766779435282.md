# Funnels (Workflow-Driven Ingress & Orchestration)

## 1️⃣ Purpose
Visual observability and governance over inbound data flows. Manages how external data (GHL) enters the system and routes to CRM objects.

## 2️⃣ What This Tab IS
*   A monitoring dashboard for Ingress instances.
*   A configuration screen for mapping GHL Webhooks to Neo8 logic.
*   A log viewer for ingress events.

## 3️⃣ What This Tab IS NOT (Scope Locks)
*   ❌ A webpage builder (Use GHL).
*   ❌ An ad manager (Use FB/Google).
*   ❌ A logic executor (The UI monitors; Neo8 executes).

## 4️⃣ UI Architecture
*   **Layout**: Dashboard Grid (Active Funnels) + Detail View.
*   **Detail View**: 
    *   **Header**: Controls & Status.
    *   **Main**: Process Visualizer (Ingress -> Logic -> Output).
    *   **Right Rail**: Event Log Stream.
*   **Drawers**: Configuration Drawer.

## 5️⃣ Core UI Elements
*   **Funnel Card**: Status (`Active`/`Error`), Volume Sparkline, Source Badge.
*   **Visualizer Nodes**: Graphic representation of the data path.
*   **Log Row**: Timestamp, Event Type, Payload Preview, Outcome (`Success`/`Blocked`).

## 6️⃣ User Interaction Flow
1.  User clicks "Instantiate Funnel".
2.  User selects "Source" (e.g., GHL Form).
3.  User configures "Logic Path" in Drawer (e.g., Dedupe -> Create Lead).
4.  User clicks "Activate".
5.  System generates Webhook URL.
6.  User copies URL to GHL.
7.  User clicks "Test Handshake" to verify connectivity.

## 7️⃣ Data Model (Conceptual)
*   **Funnel_Instance**: `id`, `name`, `webhook_url`, `status`, `config_json`.
*   **Ingress_Event**: `id`, `funnel_id`, `payload`, `outcome`, `timestamp`.

## 8️⃣ Workflow & Neo8 Integration
*   **Ingress**: External POST -> Webhook -> Neo8.
*   **Processing**: Neo8 applies Logic defined in `Funnel_Instance`.
*   **Output**: Neo8 updates CRM DB. UI reflects log.

## 9️⃣ Permissions & Governance
*   **View**: All Roles.
*   **Configure/Edit**: Architects Only.
*   **Pause/Resume**: Dispatchers, Architects.

## 🔟 Explicit Edge Cases
*   **Webhook Flooding**: Rate limiter active. Logs show "Rate Limit Exceeded".
*   **Schema Mismatch**: If GHL payload changes, validation fails. Funnel status -> `Error`.
